# UI_SPEC — 위젯 UI & 윈도우 컨트롤

> 프레임 없는 데스크톱 위젯의 창 동작, 컨트롤 버튼, 레이아웃, 상태 표시 명세.
> 관련 원본: `docs/TEMP_SPEC.md` L16(X 종료), L17(최소화/최대화), L18(플러스), L19(좌/우)

## 1. 개요 & 범위

위젯의 **화면 표현과 윈도우 동작**을 다룬다. 버튼이 트리거하는 로직은 다른 문서를 참조한다.
- 플러스/좌우 버튼의 **동작 로직** → `CONNECTION_SPEC.md`
- 데이터 그리드의 **데이터·상태** → `DATA_SPEC.md`

## 2. 화면 구조 (Layout)

```
┌──────────────────────────────────────────────┐
│ [별칭▾]  ◀  ▶        +     ─   □   ✕          │  ← 컨트롤 헤더바 (드래그 영역)
├──────────────────────────────────────────────┤
│            claude     codex     gemini         │
│  일일      $/토큰     $/토큰    $/토큰          │  ← 데이터 영역 (2×3, DATA_SPEC)
│  월        $/토큰     $/토큰    $/토큰          │
├──────────────────────────────────────────────┤
│  마지막 갱신: 00:00:00            ● 연결됨      │  ← 상태 푸터
└──────────────────────────────────────────────┘
```

## 3. 구현 단계 (Implementation Steps)

### 3.1 윈도우 셸
- `BrowserWindow`: `frame:false`, `transparent:true`, `alwaysOnTop:true`, `skipTaskbar:true`
- 헤더바에 `-webkit-app-region: drag`로 위젯 이동, 버튼류는 `no-drag`
- 위치/크기/최소화·최대화 상태를 `electron-store`에 영속화(재시작 복원)

### 3.2 컨트롤 헤더바
- 우측: **─(최소화)**, **□(최대화)**, **✕(종료)** 버튼
- 좌측: 현재 호스트 **별칭** 표시, **◀ / ▶**(좌/우 전환), **+**(등록)
- 모든 버튼은 클릭 영역 확보 + hover 피드백

### 3.3 종료 (✕)
- 위젯 종료. (트레이 옵션이 있다면 종료 vs 트레이 수납 정책 정의 — 기본: 앱 종료)

### 3.4 최소화 (확정: 접기 — 헤더바만 남김)
- 데이터 영역·푸터를 숨기고 **헤더바만** 남겨 창 높이를 축소
- 다시 누르면 펼침. 접힘 상태를 영속화
- (OS 작업표시줄 최소화가 아니라 위젯 자체 "접기")

### 3.5 최대화 (확정: 상세 확장)
- 창을 키우며 **더 많은 정보**를 노출(예: 일별 추이, 모델별 세부 breakdown, input/output 분리)
- 다시 누르면 기본 크기로 복귀. 확장 상태를 영속화

### 3.6 플러스 버튼 (+) — IP 등록 폼
- 클릭 시 등록 폼/모달 표시: host, port, username, 인증(키 경로 또는 비밀번호), alias
- 제출 → `host:add`(연결 테스트 포함). 로직·검증은 `CONNECTION_SPEC.md`
- 성공/실패 결과를 폼에 표시

### 3.7 좌/우 버튼 (◀ ▶) — 호스트 전환
- ◀ 이전 / ▶ 다음 호스트로 순환 전환 → `host:switch`
- 현재 호스트의 **별칭**을 헤더에 표시(등록된 호스트가 없으면 "등록된 호스트 없음" + 등록 유도)
- 전환 즉시 데이터 재조회(DATA_SPEC)

### 3.8 데이터 영역 + 상태 표시
- 2×3 그리드 렌더링(DATA_SPEC의 `usage:update` 수신)
- 셀: 데이터 없음 → "없음"
- 연결 실패 → 데이터 영역 전체 "연결 안됨"으로 대체
- 푸터: **마지막 갱신 시각**, **연결 상태 인디케이터**(연결됨/끊김), 로딩/에러 표시

### 3.9 렌더러 ↔ 메인 연동
- preload `contextBridge`로 노출된 API만 사용
  (`window.host.*`, `window.usage.onUpdate`, `window.widget.minimize/maximize/close` 등)
- Node 직접 접근 금지 (CLAUDE.md 보안 규칙 준수)
