# UI_SPEC — 위젯 UI & 윈도우 컨트롤

> 프레임 없는 데스크톱 위젯의 창 동작, 컨트롤 버튼, 레이아웃, 상태 표시 명세.
> 관련 원본: `docs/TEMP_SPEC.md` L16(X 종료), L17(최소화/최대화), L18(플러스), L19(좌/우)

## 1. 개요 & 범위

위젯의 **화면 표현과 윈도우 동작**을 다룬다. 버튼이 트리거하는 로직은 다른 문서를 참조한다.
- 플러스/좌우 버튼의 **동작 로직** → `CONNECTION_SPEC.md`
- 데이터 그리드의 **데이터·상태** → `DATA_SPEC.md`

## 2. 화면 구조 (Layout)

```
┌──────────────────────────────────────────────┐
│ ● [별칭]  ◀  ▶        +     ─   □   ✕         │  ← 컨트롤 헤더바 (드래그 영역, 좌측 연결 점)
├──────────────────────────────────────────────┤
│            Claude     Codex     Gemini         │  ← 데이터 영역 (2×3 그리드, 스크롤)
│  일일      칸=비용     없음      없음           │     칸 = 비용 + 모델칩 + 토큰세부(Input/…/Total)
│  월간      +모델+세부  …        …              │     토큰 K/M/B 축약(전체는 hover), 넉넉한 간격
├──────────────────────────────────────────────┤
│  갱신: 00:00:00    [ccusage 없음]  ● 연결됨     │  ← 상태 푸터(갱신시각 · setup 상태 칩 · 연결)
└──────────────────────────────────────────────┘
```
배경은 **불투명**(뒤 레이아웃 겹침 방지). 헤더 좌측에 연결 점(접힌 상태에서도 표시).

## 3. 구현 단계 (Implementation Steps)

### 3.1 윈도우 셸 (트레이 상주 + 드래그 이동, Windows 1순위)
- `BrowserWindow`: `frame:false`, `transparent:true`, `alwaysOnTop:true`, `skipTaskbar:true`, `movable:true`
  - **항상 최상위 레이어**: `setAlwaysOnTop(true, 'screen-saver')` + `setVisibleOnAllWorkspaces(true, {visibleOnFullScreen:true})`
    → 전체화면 앱·다른 always-on-top 창 위에도 가리지 않음(필요 시 핫키로 숨김).
  - 창은 `transparent:true`이지만(라운드 모서리용) **위젯 본문 배경은 불투명**(`#15171c`)으로 둔다.
  - **드래그 이동**: 헤더바 `-webkit-app-region: drag`(버튼류 `no-drag`). 위치는 `windowBounds.x/y`에 저장·복원. **최초 실행만** 우측 하단.
  - **사용자 리사이즈**: `resizable:true`. 최소 `MIN_WIDTH×MIN_HEIGHT`(280×160), **최대 = 현재 모니터 작업영역의 가로·세로 절반**(`applyResizeBounds`, 이동 시 모니터 추종). 크기는 `windowBounds.width/height`에 저장·복원.
  - 콘텐츠는 창 크기를 채운다: 캐러셀 패널 = 뷰포트 100%, 에이전트 칸 = `1fr`(창 너비에 맞춰 확장/축소), 넘치면 세로 스크롤.
- **시스템 트레이(숨겨진 아이콘) 상주**: 표시/숨김 토글은 **트레이 좌클릭** 또는 **글로벌 핫키 `Cmd/Ctrl+Shift+U`**.
  - 상태는 `electron-store`(`alwaysShow`)에 영속. 창을 숨겨도 앱은 트레이에 상주(`window-all-closed`에서 종료 안 함).
  - 트레이 **우클릭 메뉴: `종료`만**(`app.quit`).
- 위치·너비·상시노출 상태를 `electron-store`에 영속화(재시작 복원).

### 3.2 컨트롤 헤더바
- 우측: **─(트레이로 숨기기)** 버튼
- 좌측: 연결 점, **◀ / ▶**(좌/우 전환), 현재 호스트 **별칭**, **+**(등록), **🗑(현재 호스트 삭제 — 내장 로컬 제외)**
- 🗑 클릭 시 확인 모달 → `host:remove`(자격증명 정리 + 다음 호스트 재선택). 버튼은 `no-drag`, 나머지 헤더는 드래그 이동 영역.

### 3.3 ✕ (숨기기 — 트레이로 수납)
- 앱을 종료하지 않고 **위젯을 숨긴다**(상시노출 OFF, `widget:hide`). 앱은 트레이에 계속 상주.
- 완전 종료는 **트레이 메뉴 '종료'**(`app.quit`)로만.

### 3.4 최소화 ─ (확정: 접기 — 헤더바만 남김)
- 데이터 영역·푸터를 숨기고 **헤더바만** 남겨 창 높이를 `COLLAPSED_HEIGHT`로 축소(토글: 접힘↔펼침)
- 접힘 상태를 영속화 (OS 작업표시줄 최소화가 아니라 위젯 자체 "접기")

### 3.5 최대화 □ (확정: 펼치기 — 단순화, 2단계 없음)
- **펼침(normal)** 한 단계만. □는 **항상 펼침** — 특히 **최소화 상태에서 데이터를 다시 보이게** 하는 동작.
- 펼침 높이는 **콘텐츠에 맞춤**: 렌더러가 헤더+데이터+푸터의 실제 높이를 측정해 `widget:fitHeight`로 전달하고,
  메인이 그 높이로 창을 리사이즈한 뒤 고정한다 → **하단의 빈 공간이 생기지 않는다**(`[SHOWN_MIN, SHOWN_MAX]` 클램프).

### 3.6 플러스 버튼 (+) — IP 등록 폼
- 클릭 시 등록 폼/모달 표시: host, port, username, 인증(키 경로 또는 비밀번호), alias
- 제출 → `host:add`(연결 테스트 포함). 로직·검증은 `CONNECTION_SPEC.md`
- 성공/실패 결과를 폼에 표시

### 3.7 좌/우 버튼 (◀ ▶) — 호스트 전환
- ◀ 이전 / ▶ 다음 호스트로 순환 전환 → `host:switch`
- 현재 호스트의 **별칭**을 헤더에 표시(등록된 호스트가 없으면 "등록된 호스트 없음" + 등록 유도)
- 전환 즉시 데이터 재조회(DATA_SPEC)

### 3.8 데이터 영역 + 상태 표시
- 렌더링(DATA_SPEC의 `usage:update` 수신): **2×3 그리드**(행=기간 일일/월간 × 열=프로바이더). 칸은 **ccusage 전체 항목**(DATA_SPEC §2.4).
- 토큰은 **K/M/B 축약**(전체 숫자 hover·우측정렬), **모델은 칩**(2개 이상이면 줄바꿈 → 깨짐 방지). 콘텐츠 간 **넉넉한 세로 간격**.
- 데이터 영역은 항목이 많아 **세로 스크롤**(헤더 행 sticky).
- 셀: 데이터 없음 → "없음"
- 연결 실패 → 데이터 영역 전체 "연결 안됨"으로 대체
- 푸터: **마지막 갱신 시각**, **의존성 상태 칩**(§3.10), **연결 상태 인디케이터**(연결됨/끊김), 로딩/에러 표시

### 3.9 렌더러 ↔ 메인 연동
- preload `contextBridge`로 노출된 API만 사용
  (`window.api.usage.onUpdate/refresh`, `window.api.host.*`, `window.api.setup.check/install/status`,
   `window.api.widget.setView/getView/close`)
- Node 직접 접근 금지 (CLAUDE.md 보안 규칙 준수)

### 3.10 의존성 점검/설치 (SETUP UI) — `SETUP_SPEC.md` 연계
- **상태 칩**(푸터): 현재 선택 호스트의 의존성 상태를 색상으로 노출 — `정상`/`ccusage 없음(npx 폴백)`/`node 없음`/`설치 실패`/`미점검`.
  호스트 전환 시 캐시(`setup:status`)로 즉시 표시 후 실점검(`setup:check`)으로 갱신.
- **점검/설치 패널**: 칩 클릭 시 모달 — 현재 호스트를 점검해 OS·node/npm/ccusage(설치✓·버전/✗)·상태를 보여주고,
  누락 시 **설치 명령 안내 + [설치] 버튼**(클릭이 동의=`setup:install`) → 결과(outcomes) 표시. [다시 점검] 제공.
  로컬(내장 `local`)·원격 호스트 모두 동일하게 동작.
